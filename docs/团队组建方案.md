# 公司法务智慧管理系统 — 敏捷开发团队方案

## 一、团队角色映射

| 敏捷角色 | 对应实现方式 | 职责 |
|----------|-------------|------|
| **Product Owner (你)** | 你本人 | 提需求、验收、排优先级 |
| **Scrum Master** | QClaw 主 Agent（硅基先锋） | 拆解任务、分配子代理、协调进度、移除障碍 |
| **后端开发** | 专用子代理 (backend-dev) | API 开发、数据库、中间件、接口联调 |
| **前端开发** | 专用子代理 (frontend-dev) | 页面、组件、状态管理、UI/UX |
| **测试/QA** | 专用子代理 (qa-tester) | 编写测试用例、自动化测试、回归验证 |
| **AI/算法工程师** | 专用子代理 (ai-engineer) | AI Agent 接入、RAG、知识库、智能审查 |
| **DevOps** | Skill: github + 脚本 | CI/CD、部署、数据库运维、备份 |

## 二、团队扩编方案

### 阶段一：立即组建（当前 Sprint）

| 角色 | 实现方式 | 触发方式 |
|------|---------|---------|
| 后端开发 | `sessions_spawn` 子代理 | Scrum Master 按需派单 |
| 前端开发 | `sessions_spawn` 子代理 | Scrum Master 按需派单 |
| DevOps | 已安装的 `github` skill | 由 Scrum Master 直接调用 |

**运作模式：**
- 你提出需求 → Scrum Master 拆解为 Task → spawn 对应子代理执行 → 完成后汇报验收
- 子代理共享同一工作区，代码直接写入项目目录
- 每个子代理完成后的代码由 Scrum Master 审查后提交

### 阶段二：安装关键 Skill 增强能力

| Skill | 增强角色 | 用途 |
|-------|---------|------|
| `frontend-design` | 前端开发 | 高质量 UI 设计，避免千篇一律的 AI 风格 |
| `self-improving-agent` | 全团队 | 错误学习、持续改进 |
| `proactive-agent` | Scrum Master | 主动检查进度、预警风险 |
| `github` | DevOps | ✅ 已安装可用 |

安装命令（待你确认后执行）：
```
skillhub install frontend-design
skillhub install self-improving-agent
skillhub install proactive-agent
```

### 阶段三：定制专业 Agent（中期）

用 `another_them` skill 蒸馏专业角色，创建持久化子代理：

| 定制 Agent | 蒸馏来源 | 职责 |
|-----------|---------|------|
| **法务领域专家** | 法律知识库 + 法务系统设计报告 | 合同审查规则、法律合规校验、知识图谱构建 |
| **全栈架构师** | 系统架构设计经验 | 技术方案评审、架构决策、性能优化 |
| **项目经理** | 敏捷开发方法论 | Sprint 规划、燃尽图、风险预警 |

## 三、敏捷迭代节奏

```
Sprint 周期: 1周（5个工作日）
├── Day 1: 你提出 Sprint 目标 → Scrum Master 拆解 Task
├── Day 2-4: 子代理并行开发 → Scrum Master 协调集成
├── Day 5: 集成测试 + 你验收 → Sprint Review
└── 滚动进入下一 Sprint
```

## 四、你的参与方式

| 场景 | 你的动作 |
|------|---------|
| 提需求 | 直接说，如"加个案件管理" |
| 验收 | 我给你演示截图/录屏，你说通过或不通过 |
| 改方向 | 随时打断，说"停，改做XX" |
| 查进度 | 问"当前进度"→ 我返回 Sprint 燃尽图 |

**你不做的：** 写代码、调 Bug、管 Git、装环境——全部由团队完成。
